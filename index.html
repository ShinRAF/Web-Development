What's this website for?

This is for?

This website is made to be a want-to be newspaper and an area to see latest information about the world of technology, gaming, internet and many more this was also made for the purpose of an assignment for Website Development. This website has 5 pages, this pages are: Homepage(Landing Page), Reviews, Jobs, Tutorial and Glossary. I will be talking about these pages in the aims and objectives part!. This is for 50% of the grade for Web development so I hope I did a good job.I will be trying to get news articles from games and technology from this time and throw in intresting articles. While I don't have a page for each article, I will be linking the source for each of them! This is so I don't get plagirised.

I hope you find this page interesting and pleasing!

About me?

Hi, my name is Stephen. I am very new to the world of coding and website development, and this is a really big step for me as an individual. As I'm still learning, the code is very rough and to be honest some of it is guess work! I have coded this website by trial and error. I've watched youtube videos, read notes and lectures to learn HTML, and as I thought I have a very long road ahead of me in the world of coding. I have coded the CSS mostly by myself, but I have taken some of the Navigation and columns as a guideline.

I have used one of Lizs(Web development tutor)navigation example from class, and one of her sample newspapers. For navigation it was week 6 and I borrowed the general CSS of it but took out some parts and changed things around to make it my own. For the coloumns I found a project in lizs Codepen called Responsive Newspaper Example, some of the CSS looks familiar.

Aims and Objectives?

For the creation of this website, I had to follow the specification from the brief, we had to follow this to achieve a grade. This website was made with specific objectives and aims, those aims are:

  • Build a newspaper - The layout will need to be in a column style and it needs to be responsive webiste with 5 or more pages.
  • Develop your code idividually - the code needs to be original, we are allowed to use other people's code but we have to comment, reference and make it into our own.
  • The content - The content will need to be relevant to each page. We are able to work with others to lighten the load but we have a choice to work alone.
  • A report - A report is needed which will need to inlcuded the development of the website, tihs is imformal and will need wireframes and screen

Back to the pages, these pages have to be responsive and meets it's objectives. These objectives are:

  • Landing/Homepage(index.html) - Introducing the website and what to expect, including an image of the team?(I might just use a stock image from unsplash since I'm working alone. This landing page will need to the the first page when being accessed through GitHub.
  • Reviews(reviews.html - This could be a written or a podcast(audio) one or two of these reviews are needed, these reviews must be about 'current' web related resources.
  • Glossary(glossary.html) - This page will need to include images,audio or video. The page will need to have web development terms, this information can be taken from the internet but will need to be linked to prevent plagiarisim.
  • Jobs(jobs.html) - This page will have two web development jobs and will need to be compared, the structure and organisation is important here.
  • Tutorials(tutorials.html) - This page will have techniques and requires videos and some text, these videos can be done by the team or in Youtube. This will need to be referenced and high quality tutorials.
